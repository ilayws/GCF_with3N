# Output File Organization

The `xB_Q2_scan.cpp` program now automatically creates organized directories for its output files.

## Directory Structure

When you run `xB_Q2_scan`, it will create the following directory structure:

```
xB_Q2_scan_output/
├── txt_files/           # All histogram data files (.txt)
│   ├── hist_xB_Q2_region*.txt
│   ├── hist_theta12_theta23.txt
│   ├── hist_*_1D.txt
│   └── hist_*_region*_1D.txt
├── png_files/           # All plot images (.png)
│   └── xB_Q2_heatmap_*.png
└── data_files/          # Reserved for other data formats
```

## File Types by Directory

### `txt_files/`
- All histogram data files in text format
- 1D histograms for individual variables
- 2D histograms for xB-Q2 correlations
- Region-specific histograms
- Angle correlation histograms

### `png_files/`
- PNG images generated by the Python plotting script
- Automatically created when running `xB_Q2_heatmap_plot.py`

### `data_files/`
- Reserved for future use (binary data, other formats)

## Backward Compatibility

The Python plotting script (`xB_Q2_heatmap_plot.py`) has been updated to:
1. First look for files in the organized directory structure
2. Fall back to the current directory if the organized structure doesn't exist
3. Automatically create the `png_files` directory when saving plots

## Usage

1. Run the C++ program: `./xB_Q2_scan [n_events] [theta_bins] [use_CM]`
2. Run the Python plotting script: `python xB_Q2_heatmap_plot.py`

The programs will automatically handle directory creation and file organization.
